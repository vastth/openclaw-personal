# OpenClaw 审查标准

**统一审查标准。** Copilot 自查时参照，Claude Code `/oc-review` 时作为评判依据。

---

## 1. 安全性（必须通过）

参见 `security-checklist.md`，三级清单全部过关。

## 2. 架构一致性

- 新增配置结构与 `openclaw.json` 现有风格一致（camelCase、注释风格、嵌套层级）。
- 新增文件在正确目录下（配置 → 项目根/对应模块，文档 → `docs/`，规则 → `.claude/rules/`）。
- 跨文件改动无遗漏（改了配置是否同步了文档？改了规则是否同步了 CLAUDE.md？）。

## 3. 功能正确性

- 关键路径有最小验证记录（至少 1 条验证命令 + 结果）。
- 配置变更后 JSON5 可解析。
- 不引入明显回归（新改动不破坏已有功能）。

## 4. 代码质量

- 函数 ≤50 行，文件 ≤400 行（最大容忍 800 行）。
- 嵌套深度 ≤4 层。
- 无注释掉的废弃代码块（应删除而非注释）。
- 无临时/调试变量提交。

## 5. 文档完整性

- 新增功能/配置有对应文档或注释。
- handoff 记录完整（改动文件、验证结果、风险、下一步）。

---

## 审查结论标准

| 结论 | 条件 |
|---|---|
| **PASS** | 无 Critical/High，Medium 已记录 |
| **PASS WITH NOTES** | 无 Critical，High 问题已知晓且风险可接受 |
| **FAIL** | 存在 Critical，或 High 问题未说明 |
